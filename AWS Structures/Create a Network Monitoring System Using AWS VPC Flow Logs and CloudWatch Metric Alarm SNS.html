Create a Network Monitoring System Using AWS VPC Flow Logs and CloudWatch

Lab Details

  1. This lab walks you through the steps to Create a Network Monitoring System Using AWS VPC Flow Logs and CloudWatch.

  3. AWS Region: US East (N. Virginia) us-east-1.

Introduction 

Amazon Virtual Private Cloud

  • Amazon Virtual Private Cloud (Amazon VPC) lets you provision a logically isolated section of the AWS Cloud where you can launch AWS resources in a virtual network that you define. You have complete control over your virtual networking environment, including the selection of your own IP address range, creation of subnets, and configuration of route tables and network gateways.

  • You can use both IPv4 and IPv6 in your VPC for secure and easy access to resources and applications.

  • You can easily customize the network configuration of your Amazon VPC. For example, you can create a public-facing subnet for your web servers that have access to the internet.

  • You can also place your backend systems, such as databases or application servers, in a private-facing subnet with no internet access. You can use multiple layers of security, including security groups and network access control lists, to help control access to Amazon EC2 instances in each subnet.

Task 7: Test the CloudWatch Alarm

In this task, you will test the application by sending an ICMP request, i.e., ping the google server and verify whether the CloudWatch alarm sends out an email or not.

  1. Make sure you are in the N.Virginia Region.

  2. Navigate to  at the top and click on  under .

  3. Click on the Instance Id of EC2Instance and then click on

  4. Now in EC2 Instance Connect tab, click on .

  5. In a new tab, it will open the terminal window.

  6. Now, lets send some traffic over the ICMP protocol by ping google.com.

    • Type the command ping google.com on the terminal window and press Enter. 

    • Since the NACL has a deny rule for the outbound ICMP traffic, you will not receive any response. Enter Ctrl+C to exit.

  1. This would result in a REJECT event for the CloudWatch Metric Filter that would trigger the CloudWatch Alarm. 

  2. Navigate to the CloudWatch service in the AWS Management Console and click on the All Alarms on the left side panel.

  3. Select the alarm CloudWatch-Alarm-Ping-Deny to view the details. 

  4. You may need to wait for a few minutes for the alarm state to change to In alarm.

  1. Also, you will receive an email with details of the alarm.

  1. Hence, CloudWatch Alarm has been tested successfully.
